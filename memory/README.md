# 记忆目录

这个目录只保留记忆格式说明。正式运行时，本地轻量个人决策记忆默认存放在当前 workspace 或本地仓库下的 `info-alchemist/memory/`，避免把用户数据写进 skill 安装包。

## 文件

- `alchemy_records.jsonl`：用户在完整信息炼金报告后明确表达决策时生成一条 Alchemy Record
- `personal_voi_profile.md`：根据记录重新生成的个人决策画像，包含近期决策记录、惯性决策模式、有效证据类型、最近决策洞察和下次询证提醒

## 个人决策画像

`personal_voi_profile.md` 是面向用户阅读的个人决策画像，不是原始搜索日志。它的目标是帮助用户理解：

- 自己通常被什么信息触发
- 原本倾向采取什么默认动作
- 哪类证据会改变行动
- 最终选择了什么动作
- 下次询证时应该优先确认什么、避免什么低价值路径

画像根据 `alchemy_records.jsonl` 重新生成。删除或修改记录后，重新运行 `scripts/update_personal_voi_profile.py` 即可刷新。

### 顶部元信息

| 字段 | 定义 | 生成规则 |
|---|---|---|
| 最近更新 | 最近一条决策记录的日期 | 取所有记录里的最大 `date`；没有记录时取当天 |
| 已记录决策 | 当前画像使用的决策记录总数 | `alchemy_records.jsonl` 中有效 JSON 行数量 |
| 画像成熟度 | 当前画像的稳定程度 | 少于 3 条为低，3-4 条为中，5 条及以上为高 |
| 画像说明 | 对画像成熟度的解释 | 记录少时只记录线索，不生成稳定偏差判断 |

### 近期决策记录

展示最近 10 条决策记录，用于回看“触发 -> 默认动作 -> 转折 -> 最终动作”的链路。

| 字段 | 定义 | 来源 / 生成规则 |
|---|---|---|
| 日期 | 决策发生日期 | `date` |
| 场景 | 本次决策所属场景 | 优先用 `scene`；否则从 `user_query` 和 `decision_context` 推断，例如 AI 趋势机会、参考对象调研、产品机会判断 |
| 触发因素 | 用户为什么开始查这件事 | 优先用 `information_trigger` / `trigger_reason_note`；否则从 query 推断为最新变化、参考对象、机会判断等 |
| 原始动作 | 如果不继续判断，用户原本最可能采取的动作 | 优先用 `default_action`；否则按场景推断，如继续观察最新动态、继续收集更多名单、继续收集更多信息 |
| 决策转折 | 证据如何改变行动 | 优先用 `decision_turning_point`；否则根据最终状态和场景生成一句话判断 |
| 最终动作 | 用户明确表达的最终选择 | 优先用 `final_action` / `user_decision_raw`；否则用 `final_status` 映射为放弃、归档、观望、小步验证、执行 |
| 证据来源 | 本次判断参考的外部来源索引 | 优先用 `evidence_sources`，最多展示 2 条标题链接；不展示正文摘要 |

### 惯性决策模式

聚合同类记录，显示可能正在形成的个人决策模式。

| 字段 | 定义 | 生成规则 |
|---|---|---|
| 模式线索 | 重复出现的决策倾向 | 优先用 `pattern_clue`；否则按 query 归类，例如趋势类查询、参考类调研、机会判断 |
| 出现次数 | 该线索出现了多少次 | 同一模式线索下的记录数量 |
| 首次出现 | 该线索最早出现日期 | 该组记录的最小 `date` |
| 最近出现 | 该线索最近出现日期 | 该组记录的最大 `date` |
| 置信度 | 该线索的可靠程度 | 1-2 次为低，3-4 次为中，5 次及以上为高 |
| 当前判断 | 是否升级为稳定模式 | 1 次只记录线索，2 次继续观察，3 次及以上作为候选稳定模式但仍需用户确认 |

### 有效证据类型

说明不同证据对用户行动的价值，不把公开搜索误当成最终行动证据。

| 证据类型 | 定义 | 当前展示规则 |
|---|---|---|
| 公开搜索 | 新闻、公开网页、竞品信息、行业资料 | 适合形成初步判断，但不足以证明真实行动价值 |
| 亲自体验 / 录屏 / 截图 | 用户自己体验产品路径后留下的材料 | 能判断新用户路径是否顺畅 |
| 点击 / 留资 / 转化数据 | 来自页面、按钮、等候名单、试用、购买等真实行为数据 | 能判断是否值得继续投入 |

### 最近决策洞察

展示最近记录中对用户决策方式有解释价值的 `insight`。

- 最多展示最近 5 条不重复洞察。
- 跳过空洞察和包含“待补充”的占位内容。
- 如果没有可用洞察，会显示“暂无可用洞察”。

### 下次询证提醒

给同类决策设置防偏规则，避免重复低价值动作。

| 字段 | 定义 |
|---|---|
| 护栏建议 | 下次遇到类似问题时应避免或优先执行的规则 |
| 触发条件 | 什么时候启用这条护栏 |
| 状态 | 当前是否已经稳定；目前默认为候选中 |

## 存储规则

- 只存派生洞察
- 不存原始文章
- 不存完整聊天
- 不存隐私文件
- 不存敏感数据
- 尊重 `DISABLE_MEMORY=1`
