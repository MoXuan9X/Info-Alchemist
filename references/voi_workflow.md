# VOI 工作流

1. 复述用户想查什么。
2. 推断查询背后的真实决策。
3. 列出至少两个候选行动。
4. 判断默认行动。
5. 判断决策边界：
   - 离决策较远
   - 接近决策
   - 正在决策边界
   - 已经决定
6. 定义高 VOI 问题。
7. 生成联网搜索计划。
8. 搜索公开证据。
9. 把证据映射到决策影响：
   - 无影响
   - 弱影响
   - 中等影响
   - 强影响
10. 找出最挑战默认假设的痛感证据。
11. 找出公开搜索无法回答的证据缺口。
12. 选择最终状态。
13. 推荐一个最小下一步行动。
14. 定义停止继续搜索的规则。
15. 生成记忆更新。

## 最终状态判定

| 最终状态 | 适用条件 | 下一步行动 |
|---|---|---|
| 放弃 | 公开证据直接否定关键前提，或用户成本明显高于潜在收益 | 停止继续搜索，记录放弃原因 |
| 归档 | 信息有参考价值，但短期不会改变行动 | 保存证据和缺口，等新触发条件出现 |
| 观察 | 公开证据不足，继续行动风险高，且最小实验暂时不划算 | 设定观察窗口和新证据触发条件 |
| 小范围验证 | 公开证据支持继续，但关键证据必须由用户自己的数据、访谈、假门按钮或手动测试确认 | 推荐一个低成本实验，不建议直接完整开发 |
| 执行 | 公开证据强、缺口可接受，且用户已经具备明确资源和渠道 | 给出具体执行步骤和停止条件 |

高风险领域只整理证据和缺口，不输出替用户做最终决定的建议。实时公开搜索会变化，因此最终报告不追求完全幂等；可幂等的是搜索计划来源、缓存键、run log 和记忆 record_id。

不要写普通调研报告。最终输出必须帮助用户从“继续关注”移动到“下一步行动”。
