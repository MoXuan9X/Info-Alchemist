# 证据评分规则

Info-Alchemist 不给整份报告打总分。评分只用于解释“每条高价值证据为什么值得看”，不在缺口页或下一步页展示额外分数。

统一使用 0-100 分，必须写整数：

| 分数 | 含义 |
|---:|---|
| 0-19 | 无效或强噪声，不能使用 |
| 20-39 | 很弱，只能作为背景或缺口线索 |
| 40-59 | 偏弱，需要谨慎引用 |
| 60-74 | 可用，可以支撑阶段性判断 |
| 75-89 | 强，会明显影响判断权重 |
| 90-100 | 决定性，足以改变行动或结论上限 |

## 单条证据评分

每条进入 `## 高价值证据` 的证据只展示 2 个分数，分数放在证据内容之后，Markdown 表格列名固定为：

```md
| 行动影响 | 证据方向 | 发现了什么 | 怎么改变决策 | 来源 | 证据质量 | VOI |
```

HTML 证据卡会把 `证据质量` 和 `VOI` 渲染在卡片底部。不要把这些分数提前到卡片标题上，也不要算平均分。

### 证据质量

回答：这条证据可信吗？

评判标准：

| 分数 | 标准 |
|---:|---|
| 90-100 | 官方文档、官方定价、权威机构报告、平台规则、可复核的一手数据 |
| 75-89 | 高质量行业报告、清晰署名的专家/实践者内容、可追溯的产品页或案例 |
| 60-74 | 可追溯但需要交叉验证的媒体、社区、榜单、评论、教程或二手分析 |
| 40-59 | SEO 榜单、工具目录、商业软文、摘要不完整的信息片段 |
| 20-39 | 低质目录站、内容农场、噪声社媒、明显偏题或不可追溯来源 |
| 0-19 | 没有来源、链接失效或垃圾结果 |

使用规则：

- 低于 60 的证据通常不得进入高价值证据主表，只能进入缺口、噪声说明或内部日志；如果 VOI 很高但质量低，必须明确标成弱证据。
- 官方和权威来源不自动等于高 VOI；它只说明可信。

### VOI

回答：知道这条信息会不会改变行动？

评判标准：

| 分数 | 标准 |
|---:|---|
| 90-100 | 足以改变默认行动，例如从“直接做”改成“小范围验证/不做”，或改变优先级第一名 |
| 75-89 | 明显影响候选行动排序、范围、定价、渠道或停止规则 |
| 60-74 | 会影响判断权重，但通常不能单独改变行动 |
| 40-59 | 只提供背景，不足以改变下一步 |
| 20-39 | 好奇信息，知道了也不改变行动 |
| 0-19 | 对本轮决策无影响 |

使用规则：

- 高 VOI 证据优先排序。
- VOI 不是可信度；低质量证据即使看起来很刺激，也不能给高 VOI。

## 给 AI 的报告提示词

宿主模型写最终报告时，必须按下面规则计算和展示分数：

```text
不要给整份报告写五维总分。

`## 高价值证据` 每条证据只展示 2 个单条证据评分：
证据质量、VOI。统一写成 0-100 的整数，推荐格式 `N/100`。

优先使用 evidence_pack.public_evidence[].scores：
- scores.evidence_quality -> 证据质量
- scores.voi -> VOI

如果 evidence_pack 没有分数，按以下标准人工打分：
- 证据质量：来源是否可信、可追溯、可复核；官方/权威/一手数据最高，低质目录站或不可追溯来源最低。
- VOI：知道这条信息是否会改变默认行动、候选行动排序、范围、定价、渠道或停止规则。

`## 高价值证据` 表头固定为：
| 行动影响 | 证据方向 | 发现了什么 | 怎么改变决策 | 来源 | 证据质量 | VOI |

`## 缺失的证据` 只写还缺什么证据和为什么不能直接拍板，不展示证据覆盖度表。

`## 下一步行动` 只写具体下一步，不展示 VOI × 可验证性矩阵。
```
