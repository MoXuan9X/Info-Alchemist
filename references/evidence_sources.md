# 证据来源

证据轴解决“应该查什么”；评分规则解决“每条证据能不能进入判断”。单条证据评分和覆盖度规则见 `evidence_scoring.md`。

## 公开搜索证据

先按决策类型拆证据轴，再用联网搜索查。不要只按主题搜几条泛结果。

通用证据轴：

| 证据轴 | 用途 | 典型来源 |
|---|---|---|
| 市场/趋势 | 判断需求是否增长、是否只是短期热闹 | 行业报告、权威榜单、公司发布、融资/合作动态 |
| 官方能力/限制 | 判断能力是否真实可用、有什么边界 | 官方文档、API 文档、价格页、帮助中心、产品页 |
| 成本/单位经济 | 判断毛利、免费额度、失败重试成本是否可控 | 官方定价、API 计费、积分/水印/额度、竞品套餐 |
| 竞品/替代方案 | 判断红海程度和差异化空间 | 竞品首页、定价页、替代品页、榜单/对比页 |
| 用户痛点/抱怨 | 判断用户真实不满和未满足需求 | Reddit、Product Hunt、App 评论、小红书、公众号、论坛 |
| 获客/SEO/页面机会 | 判断能否低成本分发和验证 | SERP、关键词、教程页、对比页、替代品页、模板页 |
| 平台/政策/合规风险 | 判断版权、政策、平台依赖、监管风险 | 官方政策、法律/版权解释、平台条款、监管动态 |
| 专家/实践者信号 | 判断领域内行关注什么和现有解法 | 创始人、实践者、开发者、行业作者、社区高质量讨论 |

- 官方文档
- 价格页
- API 文档
- 产品页
- 竞品页面
- 社区抱怨
- 专家/高影响力实践者观点
- 教程
- 对比页
- SEO 页面类型

如果本轮缺少关键证据轴，报告必须降级结论强度：只能说“倾向”或“建议小范围验证”，不能输出强行动建议。

## 公开搜索通常无法解决的证据

公开搜索通常回答不了：

- 用户自己的受众是否需要这个东西
- 当前 GSC 曝光和 CTR
- 真实转化意图
- 付费意愿
- 假门按钮是否有人点
- 某个具体分发渠道是否有效
- 用户自己的执行成本是否可承受

这些都应该进入证据缺口。

## 缺口渠道

用户报告使用中文渠道名。内部 JSON 可以使用以下枚举：

- `public_search`
- `user_interview`
- `analytics`
- `GSC`
- `Clarity`
- `fake_door`
- `survey`
- `manual_test`
- `internal_data`
- `waitlist`
